Low-Level Games Programming

This project explores how memory management and other useful optimisations is implemented in low-level systems using C++.

Features

Memory Management
New and Delete have been overloaded, memory is instead stored in pools of contiguous memory. when an object is removed, the position in memory is then flagged as available.
Intrinsics
The vector class has been implemented utilising Intel Intrinsics.
Threading The project has been threaded in such a way that two main processes happen in parralel.
Each Frame consists of a Render and Output Thread, While the program is rendering a new frame, the program is also writing the frame created in the previous frame to memory.
This is further threaded inside of the Render Thread so that the image is rendered in 4 Scan lines, drastically reducing the time required to render a frame.
